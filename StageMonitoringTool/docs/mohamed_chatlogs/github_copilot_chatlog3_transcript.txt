In `src/student/formulier.js` the submit button does this:

- call `saveProposal(proposal)`
- only then do `window.location.assign('/?role=wachten')`

### Why your submission can fail
If any of these happens, the redirect is skipped:

1. Required field is empty
2. Mentor email is invalid
3. Mentor email uses a blocked temporary domain
   - blocked list: `tempmail.com`, `10minutemail.com`, `guerrillamail.com`, `mailinator.com`, `yopmail.com`
4. `saveProposal()` throws because:
   - server POST fails
   - and localStorage is unavailable/blocked in your browser

### Important detail
`src/main.js` will render the waiting page for everyone if the URL is `/?role=wachten`. So the problem is not role routing, it is that `window.location.assign('/?role=wachten')` never runs for your submission.

### Most likely cause for one user only

- a validation issue on your input
- or a browser/localStorage issue preventing `saveProposal()` from completing
