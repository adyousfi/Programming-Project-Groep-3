## ✅ Fixed! Here's what was wrong:

### **Root Cause: Document Validation Flag Not Reset**
When a student sent a document back to the admin, the `document_validated` flag remained `true` from the previous validation. This caused the stage to incorrectly appear in the **"Afgecheckte studenten"** (Checked Students) section instead of **"Ontvangen"** (Received) where the admin needed to review it.

### **Changes Made:**

**1. Fixed the student document upload endpoint** backend/server.js
   - Changed the stage update to reset the validation flag:
   ```javascript
   // BEFORE
   await stage.update({ status: 'DOCUMENTGEUPLOADED' });
   
   // AFTER  
   await stage.update({ status: 'DOCUMENTGEUPLOADED', document_validated: false });
   ```

**2. Enhanced the admin stages API** backend/server.js
   - Added `stageDetails` object with start and end dates to the API response
   - This ensures dates display correctly in the admin documents section

### **How It Works Now:**
1. ✅ Admin uploads document → marked as `admin_template`
2. ✅ Student downloads and fills it in  
3. ✅ Student sends back → creates `student_submission`, resets `document_validated` to false
4. ✅ Admin sees it in **"Ontvangen"** section ready to review
5. ✅ After approval, it moves to **"Afgecheckte"** section
